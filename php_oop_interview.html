OOP Core Structure
S1What is the complete anatomy of a PHP class?Easy▾
Answer
class BankAccount
{
    // 1. Class constants
    public const INTEREST_RATE = 0.05;
    protected const MIN_BALANCE = 0;

    // 2. Static properties (shared across all instances)
    private static int $totalAccounts = 0;

    // 3. Instance properties (each object has its own)
    private string $owner;
    private float  $balance;

    // 4. Constructor (called when new BankAccount(...))
    public function __construct(string $owner, float $balance = 0)
    {
        $this->owner   = $owner;
        $this->balance = $balance;
        self::$totalAccounts++;
    }

    // 5. Instance methods
    public function deposit(float $amount): static  // fluent interface
    {
        $this->balance += $amount;
        return $this;
    }

    // 6. Static methods
    public static function getTotalAccounts(): int
    {
        return self::$totalAccounts;
    }

    // 7. Getters / Setters
    public function getBalance(): float { return $this->balance; }

    // 8. Destructor (called when object is garbage collected)
    public function __destruct()
    {
        self::$totalAccounts--;
    }
}
💡 Fluent interface: Returning $this (or static in PHP 8) from methods allows chaining: $acc->deposit(100)->deposit(50)->getBalance()
S2What is $this and when is it NOT available?Medium▾
Answer
$this refers to the current object instance. It is available inside all non-static methods. It is NOT available in static methods, closures (without binding), or global scope.
class Foo {
    public $name = 'Foo';

    public function regular() {
        echo $this->name;   // ✅ Works — $this is the Foo instance
    }

    public static function stat() {
        echo $this->name;   // ❌ Fatal: Using $this in static context
    }

    public function withClosure() {
        $fn = function() {
            echo $this->name; // ✅ PHP auto-binds $this into closures inside methods
        };
        $fn();

        $arrow = fn() => $this->name; // ✅ Arrow functions capture $this automatically
        echo $arrow();
    }
}
⚠️ Trap: Static closures (static function() {}) do NOT have access to $this even inside a method. This is a common mistake in event systems.
S3What is the difference between self::, parent::, static::, and $this->?Hard▾
Answer — The most confused topic in PHP OOP
KeywordResolves ToUse When
$this->Current object instanceInstance method/property access
self::Class where code is WRITTEN (compile-time)Static access within same class
parent::Direct parent classCall parent's overridden method
static::Class that was CALLED (runtime — Late Static Binding)Factory methods, singleton inheritance
class Base {
    public static function createSelf(): static  {
        return new self();    // Always creates Base — even if called on Child!
    }
    public static function createStatic(): static  {
        return new static();  // Creates whatever class was called (LSB)
    }
}

class Child extends Base {}

$a = Child::createSelf();   // instanceof Base   ← might surprise you!
$b = Child::createStatic(); // instanceof Child  ← correct behavior

var_dump($a instanceof Child); // bool(false)
var_dump($b instanceof Child); // bool(true)
🔬 Rule: Use static:: in factory methods and singletons that will be extended. Use self:: only when you WANT to lock it to the defining class. Most PHP devs get this wrong.

S5What is object cloning and what is the difference between shallow and deep clone?Hard▾
class Address {
    public function __construct(public string $city) {}
}

class User {
    public Address $address;
    public function __construct(public string $name, Address $address) {
        $this->address = $address;
    }

    // Without __clone: shallow copy — address object is SHARED
    public function __clone() {
        // Deep clone: clone nested objects too
        $this->address = clone $this->address;
    }
}

$u1 = new User('Alice', new Address('Delhi'));
$u2 = clone $u1;  // triggers __clone()

$u2->name = 'Bob';
$u2->address->city = 'Mumbai';

echo $u1->name;          // Alice (scalar — copied by value)
echo $u1->address->city; // Delhi (deep clone — independent!)
⚠️ Shallow clone trap: Without __clone(), nested objects are copied by reference — both $u1 and $u2 share the same Address object. Modifying one changes the other. This is one of the most common OOP bugs in PHP.
S6How does PHP pass objects — by reference or by value?Hard▾
Neither exactly. PHP passes objects by handle (object identifier). The variable holds a pointer to the object. Assignment copies the pointer — not the object. But passing by reference (&) copies the pointer itself.
class Box { public int $value = 1; }

// Handle copy — both point to same object
$a = new Box();
$b = $a;              // copies the handle (pointer)
$b->value = 99;
echo $a->value;        // 99 — same object, mutation visible

// Reassigning the variable (NOT the object)
$b = new Box();       // $b now points to a NEW object
echo $a->value;        // 99 — $a unaffected

// True reference — &$b is an ALIAS of $a (same variable slot)
$c = new Box();
$d = &$c;
$d = new Box();       // Now $c ALSO points to the new object
var_dump($c === $d);  // bool(true)
🔬 Memory model: PHP object variable stores a handle (integer ID). Clone creates a new object with new handle. Reference (&) makes two variables share the same storage slot.

I3What is the final keyword and when should you use it?Medium▾
// final CLASS: cannot be extended
final class Singleton {
    private static ?self $instance = null;
    private function __construct() {}
    public static function getInstance(): static {
        return self::$instance ??= new self();
    }
}

class BadIdea extends Singleton {} // ❌ Fatal: cannot extend final class

// final METHOD: cannot be overridden by child
class Payment {
    final public function process(): void {
        // Critical flow — no child should change this
        $this->validate();
        $this->charge();
        $this->notify();
    }
    protected function validate(): void {} // children CAN override helpers
    protected function charge(): void {}
    protected function notify(): void {}
}
💡 Template Method Pattern: Seal the algorithm in a final method, but let children override the steps. This is one of the most common real-world uses of final in PHP frameworks.
I4What is the Liskov Substitution Principle in PHP?Hard▾
LSP: if S is a subtype of T, objects of type S must be substitutable for objects of T without altering the correctness of the program. Violations are subtly dangerous.
class Rectangle {
    public function __construct(
        protected int $w, protected int $h
    ) {}
    public function setWidth(int $w)  { $this->w = $w; }
    public function setHeight(int $h) { $this->h = $h; }
    public function area(): int { return $this->w * $this->h; }
}

// ❌ LSP VIOLATION: Square extends Rectangle but breaks assumptions
class Square extends Rectangle {
    public function setWidth(int $w)  { $this->w = $h = $w; } // also sets h!
    public function setHeight(int $h) { $this->w = $h = $h; } // also sets w!
}

function testArea(Rectangle $r): void {
    $r->setWidth(5);
    $r->setHeight(4);
    assert($r->area() === 20); // Passes for Rectangle
                              // FAILS for Square! (25, not 20)
}

// ✅ Fix: separate classes, share an interface instead
interface Shape { public function area(): int; }

// Abstract class: IS-A relationship + shared behavior
abstract class Logger {
    private array $buffer = [];

    abstract protected function write(string $msg): void;

    public function log(string $msg): void {
        $this->buffer[] = $msg;
        $this->write('[LOG] ' . $msg);  // calls child implementation
    }
}

// Interface: CAN-DO relationship + contract
interface Loggable { public function log(string $msg): void; }
interface Serializable { public function serialize(): string; }

// A class can do both (multiple interfaces)
class FileLogger extends Logger implements Loggable, Serializable {
    protected function write(string $msg): void {
        file_put_contents('app.log', $msg . PHP_EOL, FILE_APPEND);
    }
    public function serialize(): string { return json_encode([]); }
}
A2Can an interface extend another interface? Can it extend multiple?Medium▾
// YES — interfaces can extend multiple other interfaces
interface Readable  { public function read(): string; }
interface Writable  { public function write(string $d): void; }
interface Seekable  { public function seek(int $pos): void; }

// Stream combines all three (interface extending multiple interfaces)
interface Stream extends Readable, Writable, Seekable {
    public function close(): void;
}

// Implementing class MUST implement ALL methods from Stream + parents
class FileStream implements Stream {
    public function read(): string   { return ''; }
    public function write(string $d): void {}
    public function seek(int $pos): void {}
    public function close(): void     {}
}
💡 Design insight: Interface segregation — keep interfaces small and focused. Stream shows how to compose them when you need a combined contract. Classes can also implement each interface independently for more flexibility.

Magic Methods — Complete Reference
M1List ALL PHP magic methods with practical examplesMedium▾
class MagicDemo {
    private array $data = [];

    // __get: called when reading inaccessible/undefined property
    public function __get(string $name): mixed {
        return $this->data[$name] ?? null;
    }

    // __set: called when writing to inaccessible/undefined property
    public function __set(string $name, mixed $value): void {
        $this->data[$name] = $value;
    }

    // __isset: called when isset()/empty() on inaccessible property
    public function __isset(string $name): bool {
        return isset($this->data[$name]);
    }

    // __unset: called when unset() on inaccessible property
    public function __unset(string $name): void {
        unset($this->data[$name]);
    }

    // __call: called when invoking inaccessible instance method
    public function __call(string $name, array $args): mixed {
        if (str_starts_with($name, 'get')) {
            $prop = lcfirst(substr($name, 3));
            return $this->data[$prop] ?? null;
        }
        throw new \BadMethodCallException("Method {$name} not found");
    }

    // __callStatic: called when invoking inaccessible static method
    public static function __callStatic(string $name, array $args): mixed {
        return new static(...$args);  // factory pattern
    }

    // __toString: called when object used as string
    public function __toString(): string {
        return json_encode($this->data);
    }

    // __invoke: called when object used as function
    public function __invoke(string $key): mixed {
        return $this->data[$key] ?? null;
    }

    // __debugInfo: controls var_dump() output
    public function __debugInfo(): array {
        return ['keys' => array_keys($this->data)]; // hide values
    }
}

$obj = new MagicDemo();
$obj->name = 'Azad';        // __set
echo $obj->name;           // __get → 'Azad'
echo $obj->getName();      // __call → 'Azad'
echo $obj;                 // __toString → {"name":"Azad"}
echo $obj('name');        // __invoke → 'Azad'
echo isset($obj->name);  // __isset → true
M2__sleep(), __wakeup(), __serialize(), __unserialize() — serialization lifecycleHard▾
class DbConnection {
    private ?\PDO $pdo = null;
    public function __construct(private string $dsn) {
        $this->pdo = new \PDO($dsn);
    }

    // PHP 7 style: return property NAMES to serialize
    public function __sleep(): array {
        return ['dsn']; // Don't serialize $pdo (resource/object)
    }

    // Called after unserialize() — rebuild state
    public function __wakeup(): void {
        $this->pdo = new \PDO($this->dsn); // Reconnect
    }

    // PHP 8.1 style: return data to serialize (replaces __sleep)
    public function __serialize(): array {
        return ['dsn' => $this->dsn]; // key-value data
    }

    // PHP 8.1 style: restore state from serialized data
    public function __unserialize(array $data): void {
        $this->dsn = $data['dsn'];
        $this->pdo = new \PDO($this->dsn);
    }
}
⚠️ Security warning: Never deserialize user-supplied data using PHP's unserialize(). It can trigger magic methods and lead to Remote Code Execution (RCE). Use JSON instead, or use unserialize($data, ['allowed_classes' => [SomeClass::class]]) to whitelist.

Static Methods, Properties & Late Static Binding
LSB1Build a correct Singleton using Late Static BindingHard▾
// ❌ Broken singleton with self:: — all subclasses share ONE instance
class BadSingleton {
    private static ?self $instance = null;
    public static function getInstance(): static {
        return self::$instance ??= new self(); // Always BadSingleton!
    }
}

// ✅ Correct singleton with static:: — each class has its OWN instance
class Singleton {
    private static array $instances = []; // keyed by class name
    protected function __construct() {}   // protected: subclassable
    private function __clone() {}         // prevent cloning

    public static function getInstance(): static {
        $class = static::class;             // late-bound class name
        return self::$instances[$class] ??= new static();
    }
}

class DbSingleton    extends Singleton {}
class CacheSingleton extends Singleton {}

$db1 = DbSingleton::getInstance();
$db2 = DbSingleton::getInstance();
$ca  = CacheSingleton::getInstance();

var_dump($db1 === $db2); // true  — same instance
var_dump($db1 === $ca);  // false — different classes
LSB2Build a fluent Builder pattern using static return typeHard▾
abstract class QueryBuilder {
    protected array $conditions = [];
    protected ?int $limitVal  = null;

    public function where(string $cond): static {  // 'static' not 'self'!
        $this->conditions[] = $cond;
        return $this;
    }
    public function limit(int $n): static {
        $this->limitVal = $n;
        return $this;
    }
    abstract public function build(): string;
}

class MySQLBuilder extends QueryBuilder {
    public function build(): string {
        $sql = 'SELECT * FROM users';
        if ($this->conditions)
            $sql .= ' WHERE ' . implode(' AND ', $this->conditions);
        if ($this->limitVal)
            $sql .= " LIMIT {$this->limitVal}";
        return $sql;
    }
}

// Because where()/limit() return 'static', chaining works AND returns MySQLBuilder
$query = (new MySQLBuilder())
    ->where('active=1')
    ->where('role="admin"')
    ->limit(10)
    ->build();

// SELECT * FROM users WHERE active=1 AND role="admin" LIMIT 10
💡 Why static return type: If where() returns self (QueryBuilder), calling ->build() after would fail because self is abstract. Returning static preserves the actual subclass type.

PHP Type System — Complete
TY1What are all return type declarations? Explain void, never, mixed, union, intersectionHard▾
class TypeExamples {
    // void: returns nothing (must not return a value)
    public function logSomething(): void { echo 'logged'; }

    // never (PHP 8.1): function NEVER returns (throws or exits)
    public function fail(string $msg): never {
        throw new \RuntimeException($msg);
    }

    // mixed: any type (same as no type hint but explicit)
    public function anything(): mixed { return null; }

    // Union types (PHP 8.0): int OR string
    public function getIdOrName(): int|string { return 1; }

    // Nullable (shorthand for Type|null)
    public function findUser(int $id): ?User { return null; }

    // Intersection types (PHP 8.1): must satisfy BOTH
    public function process(Countable&Iterator $it): void {}

    // static: return type means "the class that was called on"
    public function withName(string $name): static {
        $clone = clone $this;
        $clone->name = $name;
        return $clone;  // immutable update pattern
    }

    // Disjunctive Normal Form (PHP 8.2): combine union + intersection
    public function complex(): (Countable&Iterator)|false { return false; }
}
Type coercion — strict_types
// Without declare(strict_types=1): PHP coerces types
function add(int $a, int $b): int { return $a + $b; }

// switch: loose comparison, fall-through, no return value

    case '0':   // TRUE with ==  (loose! 0 == '0')

interface SortStrategy {
    public function sort(array $data): array;
}

class BubbleSort implements SortStrategy {
    public function sort(array $data): array {
        /* bubble sort implementation */
        return $data;
    }

        return $data;
    }
}

    public function __construct(private SortStrategy $strategy) {}

    public function setStrategy(SortStrategy $s): void {
        $this->strategy = $s;
    }

    public function sort(array $data): array {
        return $this->strategy->sort($data);
    }
}

$sorter = new Sorter(new QuickSort());
$sorted = $sorter->sort([3,1,2]);

$sorter->setStrategy(new BubbleSort()); // swap at runtime
DP2Observer Pattern — event system from scratchMedium▾
🎨 Behavioral Pattern
interface Observer {
    public function update(string $event, mixed $payload): void;
}

trait Observable {
    private array $listeners = [];

    public function subscribe(string $event, Observer $obs): void {
        $this->listeners[$event][] = $obs;
    }

    protected function fire(string $event, mixed $payload = null): void {
        foreach ($this->listeners[$event] ?? [] as $obs) {
            $obs->update($event, $payload);
        }
    }
}

class UserService {
    use Observable;

    public function register(string $email): void {
        // ... create user ...
        $this->fire('user.registered', ['email' => $email]);
    }
}

class EmailNotifier implements Observer {
    public function update(string $event, mixed $payload): void {
        echo "Sending welcome email to {$payload['email']}";
    }
}

$svc = new UserService();
$svc->subscribe('user.registered', new EmailNotifier());
$svc->register('azad@example.com'); // triggers email

T1What is the output? — Constructor, parent, and static orderHard▾
🎯 Classic Trap — Tests execution order
class A {
    public function __construct() { echo 'A'; }
    public function __destruct()  { echo '~A'; }
}
class B extends A {
    public function __construct() {
        echo 'B1';
        parent::__construct();
        echo 'B2';
    }
    public function __destruct() {
        echo '~B';
        parent::__destruct();
    }
}

$b = new B();
echo 'X';
unset($b);
Output:
B1AB2X~B~A
Constructor: B1 → parent::__construct() → A → back to B → B2 → X is echoed → unset triggers destructor: child first (~B), then parent (~A)
⚠️ Key insight: Destructors fire in REVERSE order of construction — child destructor first, then parent. Parent constructor fires in the MIDDLE of child constructor (wherever parent::__construct() is called).

class Counter {

    public function getIncrementer(): \Closure {
        return function() {
            $this->count++;
            return $this->count;
        };
    }
}

PHP Object Memory Model — How It Really Works
OM1How does PHP manage object memory internally? (zval, refcount, copy-on-write)Pro▾
PHP 7+ stores variables in zval structures. Objects are managed through an object store, separate from regular variable storage.
$a = "hello" → zval { type: IS_STRING, value: "hello", refcount: 1 } $b = $a → COW: $b points to SAME zval (refcount: 2) $b .= "!" → Write: zval is COPIED for $b (refcount back to 1 each) $obj = new Foo() → zval { type: IS_OBJECT, handle: 0x1A4 } object_store[0x1A4] → { properties, class: Foo } $obj2 = $obj → zval { type: IS_OBJECT, handle: 0x1A4 } ← same handle! Both point to SAME object — no copy, no COW $obj2->x = 5 → Mutates the shared object — $obj also sees x=5 $ref = &$obj → $ref and $obj share the SAME zval slot (alias) $ref = new Foo() → Both $ref AND $obj now point to the new object!
Garbage Collection

class Node { public ?Node $next = null; }

$a = new Node();
$b = new Node();
$a->next = $b;
$b->next = $a;  // circular reference!
unset($a, $b);   // refcounts don't reach 0 — memory leak!

// gc_collect_cycles() triggers it manually
