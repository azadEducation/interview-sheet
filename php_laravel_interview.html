Q25 What is the N+1 problem and how to fix it? ▾
Answer
N+1 occurs when 1 query loads a collection, then N additional queries load each related model.
// ❌ N+1 Problem
$users = User::all(); // 1 query
foreach ($users as $user) {
    echo $user->posts; // N queries!
}

// ✅ Eager loading fix
$users = User::with('posts')->get(); // 2 queries
💡 Tip: Use Laravel Telescope or Debugbar to detect N+1 queries.

Q38 What is the Repository Pattern in Laravel? ▾
Answer
Adds an abstraction layer between controller and model. Makes code testable and swappable.
// Interface
interface UserRepositoryInterface {
    public function findById(int $id): User;
}

// Implementation

    public function findById(int $id): User {
        return User::findOrFail($id);
    }
}

// Bind in ServiceProvider
$this->app->bind(UserRepositoryInterface::class, UserRepository::class);
